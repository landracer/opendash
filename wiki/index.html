🏁 OpenDash — Universal Racecar Dashboard

A modular, bleeding-edge digital dashboard system for race cars

About OpenDash

OpenDash is an open-source, modular racecar dashboard system with four ESP32-S3 display nodes communicating over a shared I2C bus. It presents real-time engine, GPS, IMU, and battery data to the driver.

System Components

  • Center Display: 4.3" main dash — I2C master, OBD2/CAN, data aggregator
  • Left Gauge Pod: 2.8" round gauge pod — I2C slave 0x10, oil pressure/temp
  • Right Gauge Pod: 2.8" round gauge pod — I2C slave 0x11, boost/AFR
  • GPS / Telemetry Unit: 1.75" AMOLED — I2C slave 0x12, GNSS + IMU + parachute

Communication Protocol

All OpenDash nodes communicate over a shared I2C bus at 400 kHz (Fast Mode). The Center display is always the I2C master. All other nodes (Left, Right, GPS, and external BMS) are slaves.

I2C Node Addresses

Node Address Description
Left Gauge 0x10 Left-side 2.8" round gauge
Right Gauge 0x11 Right-side 2.8" round gauge
GPS / Telemetry 0x12 GPS + IMU unit
BMS (rAtTrax) 0x20 External BMS logger node

Message Format

Every I2C message uses a fixed-header format for reliability:

┌──────┬──────┬────────┬────────────────┬──────────┐
│ SYNC │ CMD │ LENGTH │ PAYLOAD │ CHECKSUM │
│ 0xAA │ 1B │ 1B │ 0–248 bytes │ 1B │
└──────┴──────┴────────┴────────────────┴──────────┘

Fields:

  • SYNC - Always 0xAA — marks start of valid message
  • CMD - Command identifier
  • LENGTH - Number of payload bytes (0–248)
  • PAYLOAD - Command-specific data
  • CHECKSUM - XOR of all preceding bytes

Key Features

Display & UI

  • LVGL-based UI — Gauges, arcs, bar charts, and numeric readouts
  • Multi-page gauge system — Up to 8 configurable gauge pages
  • Min/max tracking — Session high/low displayed per gauge page
  • Shift-light blink — Arc flashes red/blue when RPM exceeds threshold
  • Configurable data views — Choose which data points appear in each screen section
  • Touch-screen support — GT911 hardware reset sequence
  • Unit conversion — °C/°F, kPa/BAR/PSI, km/h/MPH, km/mi
  • Easy background/asset swaps — Drop converted C-array images
  • Warning system — Flashing colored overlays for critical/caution alerts
  • Outlined text rendering — 4-shadow technique for readable text

Communication

  • I2C inter-node bus — All displays share data as I2C nodes
  • BMS integration — I2C node for rAtTrax BMS data
  • OBD2 support — Read any standard OBD2 PID
  • CAN bus ready — Center unit has onboard CAN for direct ECU communication

GPS & Telemetry

  • LC76G GNSS — Multi-constellation positioning
  • Predictive lap timing — Real-time delta vs. best lap
  • QMI8658 6-axis IMU — Accelerometer + gyroscope
  • Parachute deployment — Gyro-triggered safety system

Data Logging

  • SD card logging — CSV format, configurable sample rate
  • Per-session files — Automatic file naming with timestamps
  • Post-session analysis — Compatible with common data analysis tools

Pre-Flight Checklist

  • Crew task lists — Customizable per-team checklists
  • Touch confirmation — Tap to mark items complete
  • Status sharing — Checklist state shared across all nodes via I2C

Connectivity

  • WiFi mode — For OTA firmware updates and data transfer
  • BLE mode — For low-power data sync
  • Individual control — Each unit manages its own wireless independently

Customization & Extensibility

  • Data point legend — Full list of displayable values
  • Modular sensor support — Add custom sensors via I2C/SPI/ADC
  • Programmable alarms — Threshold-based warnings
  • Drag-and-drop assets — Convert images with LVGL tools
  • Display Mode System — Center display supports multiple cycling data views
